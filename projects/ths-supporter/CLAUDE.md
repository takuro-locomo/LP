# 指示書：トータルヒップサポーター LP制作プロジェクト

この指示書は Claude Code に渡して実行させるためのものです。プロジェクトフォルダ直下に `CLAUDE.md` として置くか、最初のプロンプトで「この指示書を読んで実行して」と渡してください。

---

## 0. プロジェクト概要

助野株式会社（SUKENO）の「トータルヒップサポーター」（販売：村中医療器 オンラインストア 商品コード V0300001）の販売促進用ランディングページ（LP）を **2種類** 制作する。

| | LP1（一般・施設向け） | LP2（医療機関向け） |
|---|---|---|
| ファイル | `lp-general/index.html` | `lp-medical/index.html` |
| ターゲット | 股関節に不安のある一般生活者・高齢者、高齢者施設（介護施設・デイサービス等）の購買担当者 | 病院・整形外科（医師、看護師、理学療法士、購買担当） |
| 主訴求 | 股関節の安定サポート。日常生活・運動時の装着。筋力低下や転倒への「備え」としての予防的ケア | 股関節手術後（主に人工骨頭置換術・THA後）の患者に履かせる術後サポートウェア |
| 最終CTA | 村中医療器オンラインの注文ページへ誘導 | 同左（＋医療機関向け問い合わせ導線） |

**最終誘導先URL（両LP共通のコンバージョンポイント）：**
`https://online.muranaka.co.jp/product/V0300001/`

## 1. 最初にやること（リサーチ・素材読み込みフェーズ）

コードを書く前に、必ず以下を順番に実施し、結果を `research/` フォルダにMarkdownで保存すること。

### 1-1. 製品理解
1. `assets/reference/` に置かれた資料をすべて読む。
   - 助野の製品カタログPDF（元URL: https://www.sukeno.co.jp/pdf/total-hip-supporter.pdf ）※画像ベースのPDFなので、テキスト抽出できない場合はPDFを画像化して内容を読み取ること
   - 村中医療器の商品ページ（ https://online.muranaka.co.jp/product/V0300001/ ）※JavaScript描画のページなので、取得できない場合はユーザーに商品ページのスクリーンショットまたは保存HTMLの提供を依頼すること
   - 臨床論文PDF（ScienceDirect PII: S2949705126000149）※署名付きURLは期限切れになるため、ユーザーがダウンロードしたPDFを `assets/reference/` に置く前提。無い場合はユーザーに依頼すること
2. 製品名・素材・サイズ展開・価格・機能（編み構造、着圧設計、装着方法など）・エビデンス（論文の研究デザイン、対象、結果、限界）を `research/01_product-facts.md` に整理する。**出典のない機能・効果は一切書かないこと。**

### 1-2. 競合・高CVR LP研究
以下の観点で類似LP・競合サイトを最低8件調査し、`research/02_lp-benchmark.md` にまとめる。

- 調査対象の例：バンテリンコーワサポーター、ザムスト、お医者さんのがっちりコルセット系D2C LP、ヒッププロテクター（トクダオルソテック等）、医療機器メーカーの医療従事者向け製品ページ、介護用品EC（けあ太朗・モノタロウ介護等）のLP
- 各LPについて記録する項目：
  - ファーストビュー（FV）で何を訴えているか（キャッチコピー、ビジュアル、CTA位置）
  - ページ構成（セクションの順番）
  - 信頼要素（エビデンス、監修、レビュー、受賞、導入実績）の見せ方
  - CTAの文言・色・出現頻度・マイクロコピー
  - 価格の見せ方、FAQ、返品・サイズ交換の不安解消
- 最後に「高CVR LPの共通パターン」を10項目に要約し、本プロジェクトへの適用方針を書く。

### 1-3. CVRフレームワークの整理
`research/03_cvr-framework.md` に、LP設計に使うフレームを整理する（新PASONA、AIDCA、FVの3秒ルール、1LP=1オファー原則、CTAファースト設計、スマホ最適化、ページ速度、など）。

## 2. 法令・表現ルール（最重要・全工程で遵守）

本製品は（確認できる限り）医薬品・認証医療機器としての効能効果を標榜できない衣料型サポーターである可能性が高い。**薬機法・景品表示法・医療広告ガイドラインに抵触する表現は絶対に使わない。** 製品の正確な区分（一般医療機器の届出有無など）は `research/01_product-facts.md` 作成時に必ず確認し、不明な場合は最も保守的な表現ルールを適用すること。

### NG表現（使用禁止）
- 「痛みが治る」「痛みを予防する」「転倒を防止する」「脱臼を防ぐ」など、疾病の治療・予防効果の断定
- 「必ず」「絶対」「日本一」「最高」「No.1」などの最大級・断定表現
- 「医師も推奨」等の裏付けのない権威表現
- 論文結果の誇張（対象・条件を省略した一般化）

### OK表現（推奨）
- 「股関節まわりを面でサポート」「安定感のある歩行を支える」「動きやすさをサポート」
- 「筋力低下が気になる方の毎日の備えに」「ふらつきが不安な方の運動習慣づくりに」
- 「〜が期待できます」「〜を目的に設計されています」＋出典明記
- 論文引用時は「◯◯を対象とした研究で、△△という結果が報告されています（出典）」の形式で、研究の対象と条件を必ず併記

### その他
- 各LPのフッターに「本品は医薬品ではありません。痛みや症状がある場合は医師にご相談ください」等の注意書きを入れる。
- LP2（医療機関向け）でも、適応・使用目的は論文とメーカー資料に記載の範囲を超えないこと。
- **助野・村中医療器のロゴ、製品写真、カタログ画像は現段階では無断使用しない。** 写真が必要な箇所はCSSで作るプレースホルダー（「製品写真（メーカー提供予定）」等のラベル付き）にする。これは先方への提案用プロトタイプであるため。

## 3. LP1仕様（一般生活者・高齢者施設向け）

### ターゲットペルソナ
1. 60〜80代。股関節に違和感や不安があり、外出や運動をためらいがち。家族がネットで探して購入するケースも多い（購入者＝50代の娘・息子の想定も）。
2. 高齢者施設の施設長・機能訓練指導員。利用者の歩行支援・活動量維持のための備品として検討。

### 訴求の柱
- 股関節まわりの「安定感」（面で支える、ズレにくい、衣類感覚）
- 日常生活・運動時に無理なく着けられる（下着感覚、洗える、着脱しやすい）
- 筋力低下が気になり始めた方の「動き続けるための備え」（※予防効果の断定はしない）
- 施設導入のしやすさ（サイズ展開、複数枚購入、洗濯耐性 等、事実確認できたもののみ）

### ページ構成（案。ベンチマーク結果に基づき改善してよい）
1. FV：キャッチコピー＋着用イメージ（プレースホルダー）＋第一CTA
2. 悩み共感（チェックリスト形式：「立ち上がりでふらつく」「長く歩くのが不安」等）
3. 解決コンセプト（製品の設計思想）
4. 製品特徴 3〜5点（カタログ事実ベース）
5. エビデンス（論文の紹介・条件付き表現で）
6. 使用シーン（日常・散歩・体操・施設でのレク）
7. 施設ご担当者様向けセクション（まとめ買い・導入相談導線）
8. サイズ・価格・購入方法
9. FAQ（6〜8問：サイズ選び、洗濯、着け方、医療機器かどうか、等）
10. 最終CTA＋注意書きフッター

### キャッチコピー方向性（FV用に10案作り、最良を採用。以下はトーン見本）
- 「その一歩に、安心を履く。」
- 「股関節の不安に、毎日着けられる支えを。」
- 「動き続けたいから、いつもの下着をサポーターに。」

## 4. LP2仕様（医療機関向け）

### ターゲットペルソナ
- 整形外科医（人工骨頭置換術・THA執刀医）：術後管理の選択肢として評価
- 病棟看護師・理学療法士：着脱のしやすさ、患者の受け入れやすさを重視
- 病院購買・用度課：価格、供給安定性、発注方法

### 訴求の柱
- 股関節手術後、**主に人工骨頭置換術後の患者に「履かせる」タイプの術後サポートウェア**であること
- 従来の硬性装具・外転枕との位置づけの違い（比較優良表現にならないよう、特徴の事実記述にとどめる）
- 臨床論文に基づくエビデンス提示（研究デザイン・対象・結果・限界を誠実に記載）
- 病棟オペレーション面の利点（着脱手順、洗濯・交換運用など、資料で確認できたもの）

### ページ構成（案）
1. FV：「人工骨頭置換術後の新しい選択肢」系の医療者向けトーン＋資料請求/注文CTA
2. 臨床背景（術後管理の課題を中立的に整理）
3. 製品コンセプトと構造
4. エビデンス（論文サマリー：図表は自作の模式図で。原文図表の転載はしない）
5. 使用方法・装着手順
6. 導入の流れ（村中医療器経由の発注方法）
7. FAQ（適応の考え方、サイズ、滅菌の要否、コスト等）
8. 最終CTA（注文ページ＋医療機関向け問い合わせ）

### トーン
一般向けLPと明確に変える。感情訴求は最小限、データと運用性の説明を中心に。学会・論文引用形式は正確に。

## 5. 技術仕様

- 静的サイト。1LPにつき `index.html` ＋ `style.css` ＋ 最小限の `script.js`（フレームワーク不使用。理由：先方サーバーへの納品しやすさ）
- レスポンシブ必須。**モバイルファーストで設計**（一般向けLPは特にスマホ閲覧が主）。ブレークポイント：375px / 768px / 1200px で崩れがないこと
- フォント：Google Fonts の日本語フォント（Noto Sans JP ＋ 見出しにもう1書体）。ただしLP1は高齢者の可読性を優先し本文16px以上・行間1.8以上・コントラスト比4.5:1以上
- CTAボタンは追従表示（スマホ下部固定）を検討
- OGP・meta description・構造化データ（FAQPage、Product）を実装
- 画像はセクション5-2の生成ワークフローに従う。実製品の写真が必要な箇所のみCSS/SVGのプレースホルダーとし、`<!-- TODO: メーカー提供写真に差し替え -->` コメントを付ける
- Lighthouse でパフォーマンス・アクセシビリティ90点以上を目標
- 計測用に GA4/広告タグの挿入位置をコメントで明示（タグ自体はダミー）

### ディレクトリ構成
```
project/
├── CLAUDE.md（この指示書）
├── assets/reference/（ユーザーが資料を置く）
├── assets/images/（ChatGPTで生成した画像を置く）
│   └── image-brief.md（Claude Codeが出力する画像生成指示書）
├── research/
│   ├── 01_product-facts.md
│   ├── 02_lp-benchmark.md
│   └── 03_cvr-framework.md
├── lp-general/  index.html / style.css / script.js
├── lp-medical/  index.html / style.css / script.js
└── review-log.md
```

## 5-2. 画像・図版の生成ワークフロー（ChatGPT画像生成の活用）

LPに必要な図・イラスト・イメージ画像は、**ChatGPTの画像生成に指示を出して作成し、適宜LPに組み込む**。実際の生成はユーザーがChatGPT側で行うため、Claude Codeの役割は「何をどう生成させるかの指示書を書くこと」と「生成された画像を最適な形で組み込むこと」。

### 手順
1. 両LPのコーディング時、画像が必要な箇所をすべて洗い出し、`assets/images/image-brief.md` に一覧化する。各画像について以下を記載すること：
   - ファイル名（例：`lp1-hero-walking.png`）／使用箇所（LP1のFV等）／推奨アスペクト比とサイズ
   - **ChatGPTにそのまま貼り付けられる生成プロンプト**（日本語で具体的に。被写体、構図、年齢層、表情、背景、光、スタイルまで指定）
   - トーン統一のため、全プロンプト共通の接頭辞を定義する（例：「明るく清潔感のある医療・ヘルスケア系LP用。柔らかい自然光、白〜淡いブルー基調、日本人、実写風／またはフラットイラスト調で統一」→ LPごとにどちらかに統一）
2. ユーザーがChatGPTで生成した画像を `assets/images/` に保存する。**生成画像（またはChatGPTの共有リンク）は後日ユーザーから提供されるため、それまでは同名のプレースホルダーで実装を進めること。**
3. Claude Codeは保存された画像を確認し、WebP変換・圧縮・適切なwidth/height指定・alt属性（アクセシビリティ配慮の日本語）を付けて組み込む。画像が未生成の間は同名のプレースホルダーで進行する。

### 必須画像（この3点は必ずimage-brief.mdの先頭に入れ、LPに組み込むこと）
1. **人工股関節（人工骨頭・インプラント）のイメージ図**
   骨盤〜大腿骨と人工股関節の位置関係がわかる医療イラスト調の模式図。主にLP2の臨床背景・製品コンセプトのセクションで使用（LP1のエビデンス紹介でも簡略版を使用可）。解剖学的に正確であることを最優先し、組み込み前に必ず正誤チェックする。
2. **股関節の痛み・不安に悩む人のイメージ**
   立ち上がりや歩行時に股関節に手を当てて不安げな表情の高齢者。LP1の悩み共感セクション用。深刻になりすぎない、共感を呼ぶトーンで。
3. **サポーターを履いて快適そうに過ごす人のイメージ**
   サポーターを着用して明るい表情で散歩・体操をしているシーン。LP1のFVまたは解決コンセプトのセクション用。ただし**実製品の外観を再現・模倣しない汎用的なサポーター表現**とし、必ず「※画像はイメージです」の注記を付けること（優良誤認の回避）。ビフォーアフター的な対比構成（2の画像と並べて「改善」を示唆する等）にはしないこと。

### 生成してよい画像（推奨）
- イメージ写真系：高齢者が散歩・体操をしている生活シーン、施設でのレクリエーション風景、病棟の穏やかな風景（LP2はトーン控えめに）
- 図解系：装着イメージの模式図、股関節まわりの簡易解剖イラスト、サポート範囲を示す模式図、導入フロー図、論文結果を要約した自作グラフ（数値は原著に忠実に）
- アイコン・あしらい：特徴セクション用アイコン、チェックリスト用イラスト

### 生成してはいけない画像（禁止）
- **実製品に見える「製品写真風」画像**（実物と異なる外観を製品として見せると景表法上の優良誤認リスク。製品写真はメーカー提供まで必ずプレースホルダー）
- 実在の人物に見える「医師の推薦」「患者の体験談」風のポートレート（架空の権威づけ・体験談の捏造にあたる）
- 論文の図表の複製・トレース（引用許諾の範囲外。要約グラフは自作デザインで作り直す）
- ビフォーアフター表現（歩行が改善した等の比較画像）

### 品質チェック
組み込み前に、解剖学的な誤り（サポーターの装着位置・関節の位置がおかしい等）、指や脚の本数などAI特有の破綻がないか確認し、問題があれば修正版のプロンプトを `image-brief.md` に追記してユーザーに再生成を依頼する。

## 6. レビュー＆改善サイクル（10回・必須）

初版完成後、**両LPそれぞれについて以下のサイクルを10回転**させる。各回の内容は `review-log.md` に必ず記録すること（回数・観点・発見した問題・実施した改善・改善前後の自己採点）。

### 各回のテーマ（この順で実施）
| 回 | レビュー観点 |
|---|---|
| 1 | ファーストビュー：3秒で「誰の・何の・どうなる」が伝わるか。CTAは見えているか |
| 2 | コピー：ターゲットの言葉になっているか。抽象語を具体に置換。1文を短く |
| 3 | ページ構成・ストーリー：新PASONA等に照らして流れに飛躍・重複がないか |
| 4 | モバイルUX：実機幅375pxでの表示、タップ領域44px以上、追従CTA |
| 5 | 信頼性：エビデンス提示の正確さ、出典表記、不安解消要素（FAQ・返品/交換情報）の網羅 |
| 6 | 法令チェック：セクション2のNG表現を全文走査。1語ずつ判定して結果を表に残す |
| 7 | CTA最適化：文言・配置・回数・マイクロコピー（「約1分で完了」等）・クリック後の期待一致 |
| 8 | デザイン：視線誘導、余白、色のコントラスト、テンプレ感の排除（signature要素を1つ入れる） |
| 9 | 技術品質：HTML検証、Lighthouse計測、OGP、構造化データ、読み込み速度 |
| 10 | 総合QA：初回リサーチ（research/）と突き合わせ、高CVR共通パターン10項目の充足を最終確認 |

### 各回の手順
1. 対象観点のチェックリストを作る（5〜10項目）
2. ページをスクリーンショット等で確認し、各項目を◯△×で採点
3. ×と△の項目について改善を実装
4. 再採点し、`review-log.md` に追記
5. 採点が全◯でも「さらに良くするなら」を1つ以上実施してから次の回へ
